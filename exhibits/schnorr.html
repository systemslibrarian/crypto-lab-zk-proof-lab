The prover never transmits x directly. Instead, each round picks fresh randomness r, publishes R = g^r mod p, answers challenge c with s = r + c x mod (p−1), and passes only if g^s = R y^c mod p.

For realistic groups, recovering x from transcripts is tied to the discrete log problem. Production Schnorr-style systems operate over large groups such as secp256k1 where brute force is infeasible, while this exhibit uses small numbers for transparency.

What the verifier learns: Only that the prover knows x. The response s = r + c x changes every round because r is freshly random. Even knowing s and c, computing x requires solving the discrete logarithm — infeasible for large primes. Production Schnorr uses 256-bit parameters (secp256k1, Bitcoin signatures).
